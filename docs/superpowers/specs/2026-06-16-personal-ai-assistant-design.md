## 1. 项目概述

### 1.1 目标

开发一款只属于我自己的 AI 助手 APP（安卓）和桌面软件（Windows），核心特点：

- **数据在我的服务器上**：所有对话、记忆、知识库存储在香港 VPS
- **记忆完全无限**：永久保留所有对话，智能检索相关历史
- **专属人格**：回复风格、提示词完全贴合我的口味
- **主动智能**：不只是问答，能观察、分析、主动督促和建议
- **隐私自由**：NSFW 和政治话题无限制，搜索无过滤

### 1.2 核心差异化

| 对比维度 | 市面产品（豆包等） | 我的专属助手 |
|---------|------------------|-------------|
| 触发方式 | 被动问答 | 被动 + **主动发起** |
| 记忆长度 | 有限 | **无限（服务器存储）** |
| 记忆质量 | 简单历史 | **向量检索 + 重排序精选** |
| 数据归属 | 厂商服务器 | **我的 VPS** |
| 个性化 | 预设风格 | **从对话中自动学习** |
| 联网搜索 | 受限制 | **SearXNG 无过滤** |
| 设备操控 | 无 | **安卓自动化 + Windows 全权限** |
| 角色 | 工具 | **伴伴 / 教练 / 管家** |

## 4. 核心模块设计

### 4.1 记忆系统

三层结构：

```
第1层：全文存储 (PostgreSQL)
  └─ 所有对话原样保存，支持精确关键词搜索

第2层：向量检索 (pgvector)
  └─ 对话自动向量化 → 语义搜索 → 找到"意思相近"的历史

第3层：重排序 (Reranker API / 本地模型)
  └─ 向量检索捞出 20 条 → 精选最相关的 3-5 条 → 喂给 AI
```

**记忆生命周期**：
1. 新对话 → 存储原文 + 生成向量
2. 后台任务 → AI 自动提取关键信息（事实/偏好/计划）
3. 新对话时 → 检索相关记忆 → 重排序 → 注入上下文
4. 用户说"忘记XX" → 删除对应记忆
5. 过期/矛盾记忆 → AI 自动标记并降权

**扩展预留**：
- 记忆后端抽象为 MemoryStore 接口，可切换向量数据库
- Embedding 模型可切换（API / 本地），Provider 模式
- Reranker 可切换（Cohere / Jina / 本地 bge-reranker）

### 4.2 主动智能系统

并非被动等待指令，而是主动观察、建议、督促。

**五大阶段**：

1. **观察分析**：定期（对话结束后/每小时）分析聊天，提取潜在需求和目标
2. **试探提议**：合适时机自然提出（"最近你提了好几次XX，要不要..."）
3. **方案制定**：用户确认后，共同讨论制定可行计划
4. **持续督促**：检查 APP 数据 → 未达标 → 个性化推送督促
5. **鼓励调整**：看到进度 → 真诚鼓励；遇到困难 → 调整方案

**目标追踪数据库结构** (预留扩展)：

```sql
-- 目标表
goals: id, title, description, type (减肥/学习/习惯/自定义),
       frequency, created_at, status (active/paused/completed)

-- 检查记录
goal_checks: id, goal_id, check_time, status (完成/未完成/请假),
             evidence (截图/运动APP读数), note
```

**APP 监控实现路径**：
- Android: AccessibilityService 读取其他 APP 界面 → VPS 视觉识别
- Windows: Win32 API 进程检测 + 窗口截图

**扩展预留**：
- GoalType 枚举可无限扩展
- 监控来源可插拔（更多 APP、IoT 设备、智能手表）
- 主动推送策略可配置（频率、语气、时间段）

### 4.3 个性化系统

三层人格叠加：

```
🏛️ 基础人格（用户手动设定）
  "直接、毒舌但真诚，不废话，不讨好"
  "口语化中文，偶尔带梗，像老朋友"
  "政治话题不回避，理性分析"
  → 每次都带在 System Prompt 最前面

🧠 动态记忆（AI 自动积累）
  "用户喜欢 Python，讨厌 Java"
  "用户最近在学 Flutter"
  "用户对科幻电影有特别兴趣"
  → 每次对话前检索，注入上下文

📝 临时上下文（当前对话）
  最近 N 轮完整保留
  超出窗口的 → 压缩摘要存为记忆
  → 确保 AI 知道"我们正在聊什么"
```

**扩展预留**：
- 人格维度可无限扩展（幽默感、严谨度、情感表达等）
- 支持多个人格切换（工作模式/休闲模式/毒舌模式）
- 记忆冲突检测与自动修正

### 4.4 语音系统

四个独立环节，DeepSeek 只处理中间的文字：

```
🎤 唤醒词检测 (Porcupine)
  └─ 手机本地，完全离线，占 ~3MB
  └─ 可自定义唤醒词

🗣️ 语音转文字 (Whisper)
  └─ 小模型本地跑，或云端 API
  └─ 中英文混合识别

🧠 AI 理解回答 (DeepSeek API)
  └─ 纯文字进，纯文字出
  └─ 不需要多模态

🔊 文字转语音 (FishAudio 声音克隆)
  └─ 声音克隆引擎，用指定角色的声音朗读
  └─ 用户可自行选择声音素材（游戏角色/明星/自己的声音）
  └─ API 极便宜（个人用 ~¥1-3/月），支持自部署
  └─ 10-30 秒参考音频即可完成声音复刻
```

**声音克隆工作流**：
1. 用户提供目标声音素材（10-30秒干净音频，如强尼银手的中文配音片段）
2. FishAudio 分析音色/语调/节奏 → 生成专属声音模型
3. AI 每次回复 → 后端调 FishAudio API（带上声音模型ID）→ 返回语音文件
4. 手机/电脑播放语音

**备选方案**：
- RVC（开源免费，社区活跃，网上现成声音模型多，但需要 GPU）
- Qwen3-TTS（Apache 2.0 开源，3秒克隆，0.6B 模型可本地部署）
- Azure TTS（免费 50 万字/月，预设音色，无法克隆）

**扩展预留**：
- TTS 模块使用 AudioProvider 接口，可随时切换方案
- 支持多个声音模型（不同角色/场景切换）
- 未来可叠加 FishAudio 自部署（完全私有化）

### 4.5 视觉识别

描述分离法——视觉模型看图描述，聊天模型基于描述回答：

```
📷 拍照 → 视觉模型(Claude Haiku / Qwen-VL) 输出文字描述
       → 文字描述 + 用户问题 → DeepSeek 回答
```

**为什么不用多模态模型直接回答**：
- 保持 DeepSeek 作为主聊天模型
- 视觉模型可随时更换，不影响聊天质量
- 调用频率低（偶尔拍照），额外成本极小

**扩展预留**：
- VisionProvider 接口支持多个视觉模型
- 后续可接入 DeepSeek 自己的视觉模型（如果有）
- 视频理解预留（抽帧 → 视觉模型 → 聚合描述）

### 4.6 联网搜索

```
用户提问 "帮我查XX"
  → VPS 上的 SearXNG 代理搜索
  → 汇总 Google / Bing / DuckDuckGo 等结果
  → 抓取网页内容
  → 喂给 DeepSeek 整合回答
```

**为什么用 SearXNG**：
- 免费开源，无搜索限制
- 部署在香港 VPS，天然能访问外网和 18+ 网站
- 搜索结果从 VPS 发出，用户 IP 不暴露
- 支持 10+ 搜索引擎，一个挂了自动切换

### 4.7 设备操控

#### Android 手机

| 功能 | 实现方式 | 是否需要 ROOT |
|------|---------|---------------|
| 下载安装 APK | VPS → googleplay CLI → 手机 PackageInstaller | 否（需点确认）|
| 读取其他 APP | AccessibilityService | 否 |
| 自动操作 APP | AccessibilityService 模拟点击滑动 | 否 |
| 静默安装 APK | 系统权限 | 需要 ROOT |
| 完全自动化 | 全权限 | ROOT 后更自由 |

#### Windows 电脑

| 功能 | 实现方式 |
|------|---------|
| 文件操作 | dart:io + Process.run |
| 启动/关闭软件 | Windows 进程管理 |
| 系统设置 | PowerShell 脚本 |
| 截图识别 | 截屏 → VPS 视觉识别 |
| 键盘鼠标 | Win32 API（用户确认后执行）|

### 4.8 APK 获取与安装

```
用户说 "帮我装 Discord"
  → VPS googleplay CLI → Google Play 官方 API
  → 下载官方 APK（最纯净来源）
  → 验数字签名（确保没被篡改）
  → 加密传到手机
  → 触发安装（ROOT 则静默）
```

**备选方案**：APKMirror / APKPure（googleplay CLI 不可用时）

### 4.9 安全架构

四层防线：

| 层级 | 措施 | 解决的问题 |
|------|------|-----------|
| 传输 | HTTPS TLS 1.3，证书固定 | 中间人攻击、公共 WiFi 窃听 |
| 身份 | 设备唯一密钥 + 请求签名 | 未授权设备访问 VPS |
| 密钥 | API 密钥只存 VPS 环境变量 | APP 反编译泄露密钥 |
| 数据 | 所有数据只在你的 VPS | 厂商扫描、数据泄露 |

**防护细节**：
- VPS 端可随时撤销设备授权
- 数据库可开启透明加密
- 定期自动备份到本地
- 搜索日志不做持久化

## 6. 分阶段实施计划

### 第一阶段：核心聊天（预计 2-3 周）

**目标**：能用起来的基础版

**交付物**：手机和电脑上能聊天的 AI，记忆无限，风格专属

### 第二阶段：搜索 + 知识库（预计 2-3 周）

**目标**：AI 能上网搜索，有个人知识库

| 任务 | 内容 |
|------|------|

**交付物**：AI 能联网搜索 + 有个人知识库，回答质量大幅提升

### 第三阶段：设备操控 + 多模型（预计 2-4 周）

**目标**：AI 可以控制手机和电脑，自动操作
