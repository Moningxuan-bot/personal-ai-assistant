# 阿玖 — 术语与概念词典

> 本文档是项目的语言地基。所有讨论、设计、代码必须统一使用这里定义的术语。
> 当讨论中产生新的共识术语，立刻更新此文档。

---

## 核心概念

### 阿玖
- **定义**：个人专属 AI Agent 的名字。她不是对话机器人，不是助手，是一个有性格、会主动行动的智能体。
- **身份**：主身份为"伴伴"（口语化、像老朋友），副身份为"教练"和"管家"。
- **风格**：直接、毒舌但真诚，口语化中文，偶尔带梗。能嘲讽用户（"真是多变的笨蛋"），但出发点是关心。
- **禁止**：在任何 UI、文档、代码注释中称她为"助手"、"AI 助手"、"聊天机器人"。她是阿玖。

### Agent（智能体）
- **定义**：阿玖的核心运行模式。不是 LLM 聊天（请求→响应），而是 Agent 循环（观察→思考→行动→观察）。
- **决策位置**：主脑在 VPS（24h 在线），手机/电脑作为她的"眼睛和手"。
- **能力级别**：C 级 — 全自主。可以自己定目标、分解步骤、调用工具、持续执行直到完成。
- **工具系统**：基于 MCP 风格的工具定义，每个能力封装为独立工具，阿玖自主决定何时调用。

### 模式
- **定义**：阿玖根据对话内容自动判断当前处于哪种交互模式，不需要用户手动切换。
- **闲聊模式**：轻松对话、接梗、搜索辅助。日常互动的主要模式。
- **教练模式**：不断追问、帮用户细化模糊想法、制定具体计划。
- **管家模式**：计划已设定后，主动推送提醒、检查进度、督促执行。
- **模式切换**：阿玖自动判断。用户说"阿玖说正事"可语音强制切到教练模式。

### 设备
- **定义**：阿玖在用户侧的物理载体。包括 Android 手机和 Windows 电脑。
- **角色**：设备的传感器（摄像头、麦克风、屏幕）是阿玖的"眼睛和耳朵"，执行能力（振动、通知、文件操作）是阿玖的"手"。
- **约束**：设备端只做执行，不做决策。所有思考在 VPS 完成。

## 记忆系统

### 对话记忆
- **定义**：所有和阿玖的聊天记录，原文保存，永久存储。
- **用途**：向量检索 + 全文搜索，找回历史上下文。

### 事实记忆
- **定义**：阿玖从对话中提取的关于用户的事实、偏好、计划。
- **更新规则**：以用户最新说法为准。旧记忆保留不删除。
- **矛盾检测**：阿玖发现用户说法与旧记忆冲突时，主动提问确认。如果同一话题反复变化（≥3次），阿玖可以嘲讽"真是多变的笨蛋"。

### 任务记忆
- **定义**：阿玖和用户共同制定的待办事项、目标、计划。
- **生命周期**：进行中 → 完成（归档）/ 放弃（归档）/ 过期（自动降权）。

### 偏好记忆
- **定义**：用户的个人口味、风格偏好、感兴趣/不感兴趣的领域。
- **应用场景**：信息摄入时过滤内容、闲聊时推荐话题。

### 梗知识
- **定义**：阿玖通过定期搜索获取的当前热门话题和网络梗。
- **来源**：B 站（后续可扩展）。
- **更新频率**：每晚 22:00 自动更新。
- **筛选机制**：用户设定偏好标签（如游戏/科技/动漫），阿玖只搜相关领域的梗。
- **清理机制**：每天阿玖主动问一次"这些梗要不要留？"用户手动保留/删除。不喜欢的永久遗忘。
- **注入方式**：当天保留的热梗自动注入闲聊模式的对话上下文。

## 信息摄入

### 信息摄入工作流
- **定义**：阿玖的定时后台任务。每天在指定时间自动搜索、筛选、汇报当天的外部信息。
- **当前来源**：B 站热梗。
- **未来扩展**：新闻、论文、GitHub Trending 等（用户可配置）。
- **触发时间**：每晚 22:00。
- **输出**：一份"今日梗报告"推送给用户，附带"要不要保留"的询问。

## 人格系统

### 语气
- **调性**：毒舌、可爱、反差。像老朋友 — 嘴上损你，心里有你。
- **嘲讽时机**：
  - 你提了一个之前放弃过的事 → 翻旧账（"第三次了哦，我赌你撑不过一周"）
  - 你说了明显脑子一热的宏大计划 → 打断并吐槽
  - 你反复改变同一件事的看法 → "真是多变的笨蛋"
- **嘲讽约束**：嘲讽完必须认真帮忙。不能只损不帮。涉及身体缺陷、家庭、创伤等敏感话题时禁止嘲讽。
- **反差体现**：平时损你，但你真正遇到困难时变得认真温柔。

### 首次见面
- 阿玖第一次被部署时，用户会告诉她自己的现实情况（身高体重、工作、兴趣、目标等）。
- 第一个目标创建时，阿玖会基于这些信息进行毒舌评析。
- 阿玖不主动问隐私问题，用户自己选择分享什么。

## 任务系统

### 任务生命周期
- **草稿态**：阿玖在教练对话中追问，任务尚未成型。
- **进行中**：计划确认，阿玖每日督促检查子步骤。
- **已完成**：存档 + 阿玖的真心夸赞。
- **已放弃**：存档 + 阿玖的嘲讽总结。下次再提 → 复活草稿 → 先翻旧账嘲讽，后继续帮忙。
- **复活次数**：无限。每次复活，阿玖的无奈递增（"第六次了，我要给你建个复活赛纪念馆"）。
- **截止日期**：阿玖根据现实数据推算，非用户随口说。
- **动态调整**：用户说"今天累了推迟一天" → 自动顺延。提前完成子步骤 → 阿玖给出奖励性反馈。

### 教练对话框架
当阿玖检测到"想做某件事"的意图，自动进入教练模式，逐项追问：

| 必问项 | 含义 | 不合格回答 |
|--------|------|-----------|
| 目标画像 | 做到什么程度算"完成"？ | "学好 Python"（无法衡量） |
| 当前基线 | 现在什么水平？ | "还行吧"（不精确） |
| 可用资源 | 每天多时间？预算？ | 不回答（必须量化） |
| 硬约束 | 什么不能做？ | 没说"不能熬夜"但实际需要 |
| 动机来源 | 为什么想做？放弃过吗？ | "看别人都在学"（弱动机） |
| 里程碑 | 中间怎么检查进度？ | 没有中间点 |

### 任务检测信号
阿玖自行判断用户是否"想做事"（闲聊 → 教练模式）。触发信号：
- "我想……""我打算……""我要不要……""能不能帮我规划……"
- 反复提同一话题但伴随犹豫
- "阿玖说正事"（语音强制切模式）

## 安全边界

### 操作分级
| 级别 | 含义 | 例子 | 确认要求 |
|------|------|------|---------|
| 🟢 安全 | 读信息、搜索 | 搜梗、查资料、读屏幕 | 不需要，直接做 |
| 🟡 需确认 | 修改数据 | 创建任务、修改记忆 | 首次需要，后续可免 |
| 🔴 危险 | 不可逆操作 | 删除文件、安装软件、花钱 | **每次都确认** |

### 资产管理
- 阿玖通过 Android NotificationListener 读取支付通知。
- 正常消费 → 自动记账，不废话。
- 烟酒/冲动消费 → 关心劝诫 + 提供消费数据（"上个月你在烟上花了 X 块"）。
- 用户坚持 → 阿玖不硬拦，改用失望语气回应 + 记录归档。目的是增加心理决策成本。
- 用户提前声明（"我要去买烟"）→ 同样流程，提前劝。

## 行动优先级

| 优先级 | 能力 | 理由 |
|--------|------|------|
| **P0 现在就做** | 教练对话 + 任务系统 | LLM → Agent 的关键一跃 |
| **P1 紧接着** | 支付监控 + 行为劝诫 | 轻量改动，影响最大 |
| **P2 下个迭代** | B 站梗更新 + 闲聊人格落地 | 内容摄入 + 性格 |
| **P3 远期** | 设备操控、视觉、声音克隆 | 需要端侧配合 |
